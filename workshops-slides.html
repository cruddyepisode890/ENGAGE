What is an API?

Definition

API stands for Application Programming Interface. It's a set of rules and protocols that allows different software applications to communicate with each other.

Real-world Analogy

Think of an API like a restaurant menu. You (the client) see a list of dishes you can order (endpoints). The kitchen (backend system) prepares your order and delivers it without you needing to know how it was prepared.

Common API Examples

  • GET /api/users
  • POST /api/users
  • GET /api/products/123
  • DELETE /api/products/123

Note: APIs typically return data in JSON format, which is easy for both humans and machines to read.

HTTP Methods Deep Dive

CRUD Operations

HTTP methods correspond to CRUD (Create, Read, Update, Delete) operations in API design:

  • POST Create
  • GET Read
  • PUT Update/Replace
  • PATCH Update/Modify
  • DELETE Delete

Status Codes

HTTP responses include status codes that indicate whether a request succeeded or failed:

200 OK - Success
201 Created - Resource created
400 Bad Request - Invalid input
404 Not Found - Resource doesn't exist
500 Server Error - Something went wrong 

// Example REST API endpoint structure
const express = require('express');
const app = express();

app.get('/api/users', (req, res) => {
    // Return list of users
    res.status(200).json(users);
});

app.post('/api/users', (req, res) => {
    // Create new user
    const newUser = req.body;
    users.push(newUser);
    res.status(201).json(newUser);
});

Authentication & Security

Authentication Methods

API Keys

Simple key passed in header or URL

Authorization: ApiKey abc123...

JWT (JSON Web Tokens)

Token-based authentication that contains user info

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

OAuth

Standard for delegated authorization

/oauth/authorize?client_id=123&redirect_uri=...

Security Best Practices

  • ✓ Always use HTTPS (not HTTP)
  • ✓ Validate and sanitize all input
  • ✓ Implement rate limiting
  • ✓ Use proper authentication
  • ✓ Only expose necessary data

Practical Implementation

            
            

                

                    
Frontend API Calls

                    
                    

                        

                            
Fetch API (Native JavaScript)

                            

                                

// GET request
fetch('/api/users')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error('Error:', error));

// POST request
fetch('/api/users', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify({
        name: 'John Doe',
        email: 'john@example.com'
    })
});
                                

                            

                        

                        
                        

                            
Axios (Popular Library)

                            

                                

// GET request
axios.get('/api/users')
    .then(response => console.log(response.data))
    .catch(error => console.error('Error:', error));

// POST request
axios.post('/api/users', {
    name: 'John Doe',
    email: 'john@example.com'
});
                                

                            

                        

                    

                

                
                

                    
Backend Implementation

                    
                    

                        

                            
Node.js with Express

                            

                                

const express = require('express');
const app = express();

// Middleware to parse JSON bodies
app.use(express.json());

// Simple in-memory database
let users = [];

// Get all users
app.get('/api/users', (req, res) => {
    res.json(users);
});

// Create new user
app.post('/api/users', (req, res) => {
    const user = req.body;
    users.push(user);
    res.status(201).json(user);
});
                                

                            

                        

                        
                        

                            
Error Handling

                            

                                

app.get('/api/users/:id', (req, res) => {
    const id = parseInt(req.params.id);
    const user = users.find(u => u.id === id);
    
    if (!user) {
        return res.status(404).json({ 
            error: 'User not found' 
        });
    }
    
    res.json(user);
});
